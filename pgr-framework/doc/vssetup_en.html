PGR-framework and its folders structure

PGR-framework is a set of 3rd party libraries that we will use throughout this course to work with OpenGL (Win).
The framework contains header files, .lib, and .dll files of the following libraries:

  • FREEGLUT - The OpenGL Utility Toolkit - http://freeglut.sourceforge.net/
  • glLoadGen - OpenGL Loader Generator - https://bitbucket.org/alfonse/glloadgen/wiki/Home
  • GLM - OpenGL Mathematics Library - http://glm.g-truc.net/
  • Assimp - Open Asset Import Library - http://assimp.sourceforge.net/
  • DevIL - Image Library - http://openil.sourceforge.net/
  • pgr - PGR library - contains some useful functions

PGR-framework should be extracted into a selected folder, in this "howto" we will use h:\pgr.

Setting environment variables in Windows 7

To achieve maximum portability of the framework across wide range of platforms and users we use environment variables. Environment variable PGR_FRAMEWORK_ROOT should be set to the path to the folder containing extracted framework.

To ensure executability of your projects you have to add the path to the .dll files of the PGR-framework to the system environment variable named PATH.

Fast access to the environment variables using the "Search programs and files" box on the Start menu in Win 7.

Or you can perform the following actions: Use right mouse click Start -> Computer -> "System properties" -> "Advanced system settings" -> “Advanced” tab -> “Environment Variables …”

 

We are going to create/modify two environment variables in section "User variables for ..."

Creation of a new project in Visual studio 2010

If you are running Visual Studio for the first time then you will be aked to choose your "default enviroment settings", we recommend you to use "Visual C++ Development Settings" choice.

Type of newly created projects have to be set up to "Visual C++ Projects" -> "Win32" -> "Win32 Console Project".

Summary of all settings

  • Choose folder for the projects - e.g. H:\pgr
    • Extract the PGR-framework into its subfolder named pgr-framework
    • Setup the environment variables PGR_FRAMEWORK_ROOT to H:\pgr\pgr-framework\ and PATH
      • to %PGR_FRAMEWORK_ROOT%bin
  • Set up Visual Studio 2010 (select All Configurations):
    • C++ -> General -> Additional Include Directories: $(PGR_FRAMEWORK_ROOT)include
    • Linker -> General -> Additional Include Directories: $(PGR_FRAMEWORK_ROOT)lib
    • Linker -> Input -> Additional Dependencies: add in Debug configuration: pgrd.lib;, in Release configuration add: pgr.lib;
    • Configuration Properties -> General -> Character Set -> Not Set (disable UNICODE support) - to avoid name conflicts in DevIL library on some platforms.
